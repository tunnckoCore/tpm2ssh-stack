# Candidate Finding

## Summary
- ID: 2026-04-23-sign-ssh-F001
- Title: `ssh-add` socket validation can be bypassed with an intermediate symlink and post-validation swap
- Component: SSH agent export / socket validation
- Path(s): `src/ops/ssh.rs:159-207`, `src/ops/ssh.rs:401-476`
- Bug class: TOCTOU / symlink traversal / secret-egress redirection
- Root Cause Key: `ssh:add:resolve_socket:intermediate-symlink-socket-redirection`

## Hypothesis
- Suspected issue: `resolve_socket` rejects only a symlink at the final path component, but not symlinked ancestor directories. `add_derived_identity` validates once, then derives the private key, then uses the original raw path later. A symlink swap in between can redirect the private key to a different local socket.
- Why it may matter: `ssh-add` is the only scoped flow that intentionally exports private key material; redirecting the socket defeats the operator's intended destination and can hand the key to an attacker-controlled agent.
- Trust boundary involved: TPM/derived private key material -> local IPC peer identified by `SSH_AUTH_SOCK` / `--socket`.

## Evidence so far
- Static evidence: `resolve_socket` uses `fs::symlink_metadata(&path)` on the full path and `fs::metadata(parent)` on the parent, which follows ancestor symlinks; `add_derived_identity` stores the returned `PathBuf` before derivation and later passes the same raw path to the client.
- Dynamic evidence: local repro harness at `evidence/ssh-symlink-race/repro.rs` accepted a symlink-parent socket path, swapped the symlink during derivation, and showed the client receiving `/tmp/.../linkdir/agent.sock` while `canonicalize()` resolved to `/tmp/.../attacker/agent.sock`.
- Oracle strength: strong

## Reproduction status
- Repro available? [yes]
- Minimal trigger idea: pass `--socket` (or `SSH_AUTH_SOCK`) through a symlinked directory, validate while it points to a safe socket, then swap the symlink before `ssh-add` is invoked.
- Preconditions: PRF/seed identity with `use=ssh` + `use=export-secret`; attacker can control or race an ancestor symlink path component.

## Root cause
- Suspected root cause: final-component-only symlink rejection plus returning a noncanonical socket path that is used after a later key-derivation step.
- Confidence: high
- Competing explanations: none that fit the local repro output.
