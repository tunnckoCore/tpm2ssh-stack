# Remediation Handoff

## Finding
- ID: 2026-04-23-sign-ssh-F001
- Title: `ssh-add` socket validation can be bypassed with an intermediate symlink and post-validation swap
- Severity: High
- Confidence: High
- Root Cause Key: `ssh:add:resolve_socket:intermediate-symlink-socket-redirection`
- Path(s): `src/ops/ssh.rs:159-207`, `src/ops/ssh.rs:401-476`

## Why the fixing agent should care
- User-visible or security impact: a key that the operator intended to load into their own ssh-agent can instead be exported into an attacker-controlled local socket.
- Affected component: `tpm2-derive ssh-add` seed/PRF path.
- Trust boundary: private key material leaves TPM-only protection and crosses into another process.

## What is already known
- Repro status: reproduced locally with a custom harness that swaps an ancestor symlink after validation.
- Root cause status: confirmed; validation only checks the final component and later reuses a noncanonical path.
- Evidence quality: strong (code path + dynamic proof).

## Likely fix direction
- Checks to add or restore: reject symlinked ancestors; validate and pin the canonical socket path; optionally re-stat immediately before spawn.
- State transitions to harden: do not keep using a pre-validation raw `PathBuf` after key derivation; hand the client a verified/pinned path only.
- Bounds / lifetime / ownership rules to enforce: require every directory component on the resolved path to be owned by the current user and not group/other writable.
- Tests to add: ancestor-symlink rejection; post-validation swap attempt should fail or still target the original verified socket.

## Suggested first implementation steps
1. Refactor `resolve_socket()` to walk each path component with `symlink_metadata()` and reject symlinks before accepting the socket.
2. Return the verified canonical socket path (or equivalent pinned representation) and plumb that into `client.add_private_key()` / `ProcessSshAddClient`.
3. Add regression tests covering ancestor symlinks and a race-style swap between validation and client use.
